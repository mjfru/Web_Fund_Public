Using CSS Selectors

Setting up CSS in our HTML

The first step to effectively using CSS is to make sure we have a folder to make multiple files in. After that, create a well-named CSS title and use the < link > tag in the head of the HTML. Within that tag, use the attribute rel = "stylesheet" and href="local_url_here"

How to Start Selecting

The most basic way to select and change elements in CSS is to select all the elements. To do this, one will type the element name (i.e. h1), curly brackets spaced like JS ({}) and the desired styling contained within them.

To begin getting more specific, we can start assigning classes to elements. On this page, the first h2 element has no class attached, while the second one has a class of "selecting". The syntax for utilizing classes is: .class_name, {}

In situations where there needs to be only one of an important element, we can use an id tag instead of class. This p tag will have an id of "special" and should not be repeated elsewhere in the code.

The syntax for accessing elements with an id is #id_name, {}

Advanced Selector Usage

Wildcard

In the event that we want to style to apply to everything on our page, we can use what is known as the Wildcard selector, which has a syntax of * {}

A solid red outline is showing that utility off on this particular page...and is currently commented out because it looks atrocious! It is used though to check spacing and overall layout.

Selector Specificty

There is a hierarchy in CSS which overrides each other.

An id selector (#{}) is the strongest, followed by class (.{}), tag ({}), and finally the wildcard (*).

This means if an element has multiple tags, it will apply the style connected to the highest ranking selector.

Descendant Selector

Sometimes we know the element we want to select is inside of another element. If the "parent" element is one we can easily select (maybe it has a class on it), then we can use this to our advantage to access the child element easily.

Suppose we have a div parent element with a class of "quote". Nested within is a child p element. We could access this by using .quote p {}

Direct Descendant Selector

As its name implies, we can select a child element with this kind of selector `>`, but unlike the previous descendant selector, it will not select grandchildren.

Syntax looks like this: .header > img {style_here}...this would only change an img element directly below the parent .header. Elements nested within img would not be included.

Attribute Selector

Sometimes we have a situation where we want to style two similar elements that rely on an attribute for how they behave. The < input > element is an example of this. We do not want our input area to have the same styling as the input button. To solve this, we can use different syntax to differentiate.

input {} would select all inputs. input[type="text"]{} would select only inputs with the type "text".
